## 1. 系统概述

本系统用于 **批量管理 i茅台 预约账号**：导入账号、短信登录、定时/手动预约、中签查询、待付款提醒、代理 IP 分组等。

| 入口 | 地址 |
|------|------|
| **Web 管理端** | http://你的服务器/ |
| **API 健康检查** | http://你的服务器/api/v1/ping |
| **Android APK** | http://你的服务器/downloads/maotai-reserve.apk |

**推荐浏览器**：Chrome / Edge / Safari 最新版；分辨率建议 ≥ 1280×720。

## 2. 登录与账号角色

### 2.1 打开登录页

在浏览器地址栏输入管理端地址，进入登录界面：

![登录页](manual/screenshots/01-login.png)

| 角色 | 用户名 | 权限说明 |
|------|--------|----------|
| **最高管理员** | `owner` | 全部功能，含 **系统设置**、代理池、一键优化模板 |
| **甲方操作员** | `client` | 账号/预约/中签/任务等日常操作，**无**系统设置菜单 |

密码由部署时配置，请勿使用已废弃的 `admin / admin123`。

### 2.2 登录后界面

登录成功进入 **仪表盘**，左侧为功能菜单，右上为当前用户与退出：

![仪表盘](manual/screenshots/02-dashboard.png)

**仪表盘说明：**

- **预约尝试 / 提交成功 / 成功率**：来自「预约记录」入库统计（正式预约后才有数据）。
- **账号登录进度**：已登录与待登录数量，可跳转批量登录。
- **养号分 / 中签 / 待付款**：运营参考指标（非中签率预测）。
- **服务器定时任务**：显示下次执行时间（每日预约、中签同步、Token 巡检等）。

## 3. 推荐每日操作流程（甲方）

```text
07:00  自动 Token 巡检（异常 PushPlus 提醒）
08:30  可选：小茅运旅行定时（若已开启）
09:00  自动每日预约（含 9:05/9:10/9:15 捡漏波次）
18:03  自动同步中签 + 待付款推送
全天   App 内签到/答题/中签付款（须人工）
```

| 步骤 | 页面 | 操作 |
|------|------|------|
| 1 | 账号管理 | 确认启用账号、出口组、养号分 |
| 2 | 批量登录 | 新号发码 → 填验证码登录 |
| 3 | 商品管理 | 9 点前「从 i茅台 同步今日商品」 |
| 4 | 门店排行 | 试跑推荐选店策略（可选） |
| 5 | 任务中心 | 试跑验证 Token / 配置 |
| 6 | 中签与付款 | 18 点后同步、导出待付款、App 付款 |

## 4. 功能页面详解

### 4.1 账号管理

![账号管理](manual/screenshots/03-accounts.png)

| 功能 | 说明 |
|------|------|
| **搜索 / 出口组筛选** | 按手机号、城市、备注或 `ip001` 等出口组筛选 |
| **批量启用 / 禁用** | 勾选多行后批量操作 |
| **导入 CSV / Excel** | 支持 **CSV UTF-8**、GBK，以及 **Excel (.xlsx)**；仅「手机号」列即可，默认贵州/贵阳 |
| **导入模板（推荐）** | 项目根目录 **`账号导入模板.xlsx`**（或 `账号导入模板.csv`），删掉示例行后只填手机号，保存上传 |
| **更多列（可选）** | 见 `docs/accounts-import.example.csv`（省份、城市、出口组等） |
| **同步凭证文件** | 将数据库账号写入服务器 `credentials.json`，供 9 点预约使用 |
| **发码 / 登录 / 校验** | 单账号短信登录流程 |
| **编辑** | 选店策略、固定门店、禁用商品（中签后自动禁用会写入） |
| **养号分 / 连约天** | 运营参考，低分号优先补登录与连续预约 |

**出口组**：须与 `config.yaml` 中 `proxy_pools` 的组名一致（如 `ip001`～`ip050`），实现一 IP 多号隔离。

### 4.2 批量登录

![批量登录](manual/screenshots/04-batch-login.png)

适用于 **几十～上千账号** 集中登录：

1. 先在 **账号管理** 导入 CSV。  
2. 点击 **批量发码（全部未登录）**，按间隔（默认 90 秒）排队发短信。  
3. 各手机收到验证码后：  
   - 在列表中逐条填写；或  
   - 导出 CSV → 填好 `mobile,vcode` → **导入 CSV 批量登录**。  
4. 底部可查看任务进度与日志。

> **注意**：`转行联系方式` 这类**只有手机号**的表格，应在 **账号管理 → 导入 CSV**，不要在本页「导入 CSV 批量登录」。批量登录 CSV 必须含 **验证码** 列，示例：

```csv
mobile,vcode
13800138000,123456
```

> 发码过快易触发 429，请勿将间隔调得过小。

### 4.5 任务中心

![任务中心](manual/screenshots/07-jobs.png)

| 功能 | 说明 |
|------|------|
| **创建预约任务** | 立即执行正式预约（使用已同步凭证的启用账号） |
| **试跑** | 不提交预约，仅验证 Token、选店、加密参数 |

### 4.8 系统设置（仅 owner）

![系统设置](manual/screenshots/10-settings.png)

#### 中签优化（一键配置）

- **应用「中签优化」模板**：写入低竞争、全省、捡漏波次、推送开关、旅行定时等（见 `docs/中签优化功能说明.md`）。

- **测试 PushPlus**：验证微信推送通道。

#### 预约参数

- 申购时间、提前秒数、默认选店、全省/本市、预约模式（全部商品 / 任一成功即停）。
- 中签后自动禁用（整号或仅该商品）。
- PushPlus Token 与分类推送开关。
- 小茅运旅行 / 实验签到定时。

#### 代理池

- 维护 `ip001`～`ip050` 等与 CSV 代理对应。
- **检测全部代理**、**从账号同步出口组**、**保存** 到 `config.yaml`。

## 7. 常见问题

| 问题 | 处理 |
|------|------|
| 登录 500 / 无法登录 | 联系管理员检查 API 服务 `systemctl status maotai-api` |
| 发码失败 / 收不到短信 | 见下方 **验证码排查** |
| 预约提示无已登录账号 | 批量登录完成 → 账号管理 **同步凭证文件** |
| `secret_key 仍为占位符` | 编辑 `config.yaml` 或 `deploy/.env` 的 `MT_SECRET_KEY` 为随机串（≥32 位）；服务器部署后两者应一致 |
| Token 校验失败 Padding | `secret_key` 变更导致，需对该号 **重新短信登录** |
| 9 点商品同步为空 | 场次未开，`sessionId=0`，9:00 后再同步 |
| 中签但未付款 | App 内 24h 内支付；本系统仅提醒 |
| client 看不到系统设置 | 正常，设置仅 **owner** 可见 |

### 验证码排查（收不到短信）

1. **看 Web 提示**：点「发码」后若报错，以红色提示为准（不要只看是否收到短信）。
2. **服务器 SSL/代理（已修复一项）**  
   - 此前服务器 HTTPS 证书损坏 → 请求到不了 i茅台，**已全部失败**（已修复）。  
   - 若账号绑了 `egress_group`（如 ip001），发码会走 Glory 代理；**412 常见原因：Glory 账号未完成实名认证**（陈经理已说明），通过后代理才可用。另需：  
     - 把服务器 IP **你的服务器** 加入 **IP 白名单**；  
     - 使用「我的固定IP」导出的最新 CSV。  
   - 已修复：**发码/登录固定走服务器直连**，不再经过 Glory 代理（预约仍走代理）。若仍见 `412 Proxy Authentication`，请强制刷新页面或确认 API 已重启。
3. **429 过于频繁**（见下节「发码 429 说明」）。
4. **手机号**：须为 **真实可收短信** 的 11 位号，且已在 i茅台 App 可正常登录；虚拟号/停机号收不到。
5. **正确页面**：仅手机号 → **账号管理** 导入；发码 → **账号管理** 或 **批量登录**；不要混用「批量登录 CSV」导入只有手机号的表。

#### 发码 429 说明（重要）

**若 Web 顶部出现红色「服务器公网 IP 发码已被限流」**：说明云主机 `你的服务器` 已被 i茅台 / EdgeOne **整 IP 封禁**，在网页点发码**不会收到短信**，与手机号无关。

**可行办法（任选）：**

1. **本机发码登录（推荐）**：在 Mac/电脑上项目目录执行  
   `python scripts/local_login_to_server.py --server http://你的服务器 --username owner --mobile 11位手机号`  
   按提示输入管理密码和短信验证码，成功后 Web 上该号显示「已登」，再点 **同步凭证文件**。
2. **修复代理池（方案 B）**：详见 **`docs/方案B-Glory代理配置步骤.md`**。Glory 需先完成 **实名认证**，再加白名单 `你的服务器`；未实名时代理常 **412**。认证通过前可用 **本机发码**（`scripts/local_login_to_server.py`）。
3. **等待**：公网 IP 限流有时需数小时～一天缓解（不保证）。

---

**429** 还表示 **i茅台 认为请求太密**（限流），常见两类提示：

| 提示 | 来源 | 处理 |
|------|------|------|
| `发码间隔过短，请 XX 秒后再试` | 本系统 `login_vcode_interval`（默认 **90 秒/号**） | 等倒计时结束再点 |
| `发送过于频繁(429)，请等待 2 分钟后再试` | i茅台 接口返回 429 或文案含「频繁」 | **停手 2～5 分钟**，勿连点；系统会冷却约 **120 秒** |

**容易触发 429 的情况：**

- 同一手机号短时间多次点「发码」或批量重试；
- 批量发码间隔设得太小（建议 **≥90 秒**，号多时用 **120 秒**）；
- 多台设备/网页同时对同一批号发码；
- 此前代理/SSL 失败时反复点，成功连通后仍被平台记为高频；
- 所有号都从 **同一服务器 IP** 发码（直连），号多时要靠 **拉长间隔**，不能连点。

**建议操作：** 单号发码失败后等 **2 分钟**；批量发码用默认 90 秒或调到 120 秒；仍 429 则停 **15 分钟** 再试，或换手机 4G 热点在 App 里发一次码确认号是否正常。
