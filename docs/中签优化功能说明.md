# 中签优化功能说明

> **免责声明**：本系统优化的是「预约提交成功率、账号养号、运营效率」，**不提供 AI 中签预测**，不承诺任何中签比例。官方抽签为黑盒流程。

## P0 已实现

| 功能 | 说明 | 入口 |
|------|------|------|
| 按商品选店策略 | `items[].strategy`、`item_strategies` 模式、账号 `shop_strategy` 覆盖 | `config.yaml` / 账号编辑 |
| 养号分 | 0–100 运营参考分（连续预约、Token、出口组等） | 账号列表、仪表盘 |
| 中签后自动禁用 | `auto_disable_on_win` + `account` / `account_item` | 系统设置 |
| 定时小茅运旅行 | `travel_schedule` 按周几定时 | 系统设置 / 定时任务 `travel_batch` |
| 一键优化模板 | 写入低竞争+全省+波次+推送等预设 | 系统设置 →「应用中签优化模板」 |

## P1 已实现

| 功能 | 说明 | 入口 |
|------|------|------|
| 试跑推荐门店 | 按库存差推荐 `max_inventory` / `min_competition` | 门店排行 →「试跑推荐」 |
| 多商品优先级 | `items[].priority`、`reserve_mode: first_success` | `config.yaml` |
| 任务指定商品 | `Job.product_ids` → 仅预约选中 DB 商品 | 任务中心 |
| PushPlus 分类推送 | 新中签 / 待付款 / Token 异常 / 预约完成（可关） | 系统设置 |
| 压测报告 | 账号数、出口组、预估耗时、MySQL 建议 | 系统设置 → 压测报告 |
| 预约记录入库 | `reserve_records` 表统计提交成功率 | 预约记录 / 仪表盘 |

## MySQL 切换

```bash
# .env
MT_DATABASE_URL=mysql+pymysql://user:pass@host:3306/maotai?charset=utf8mb4

python scripts/migrate_sqlite_to_mysql.py
```

重启 API 后自动补列 `accounts` 养号字段。

## 推荐日常流程

1. 系统设置 → **应用中签优化模板**（或自行调 `min_competition` + 50 代理）
2. 账号养号分 &lt; 40 的优先补登录、旅行、连续预约
3. 9 点前任务试跑；门店页 **试跑推荐** 确认策略
4. 18:03 中签同步 + 待付款 PushPlus → App 内 24h 付款
