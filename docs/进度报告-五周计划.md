## 六、调试阶段重点关注（结合近期需求）

| 主题 | 说明 |
|------|------|
| **1000 号 + 同 Wi‑Fi** | 调试周必须上 **proxy_pools + 分组**，见 `IP隔离部署指南.md` |
| **手机端账号** | 号可在手机注册，批量请求须走代理，不能全直连 Wi‑Fi |
| **中签率** | 调试只统计 **「预约接口返回成功」**，不统计中签 |
| **合同 67%** | 建议实测后谈 **≥85% 提交成功率**（有效号、分 IP、申购时段） |

## 七、风险与需甲方配合

| 风险 | 缓解 | 需甲方 |
|------|------|--------|
| 单 IP 1000 号失败 | 多出口代理/热点 | 提供或采购代理预算、测试账号 |
| 429 / 封号 | 分批登录、错峰 | 控制发码频率，勿空验证码 |
| App 升级接口失效 | 抓包更新密钥 | 提供 1 台可抓包 iPhone 或安卓模拟器 |
| 验收标准争议 | 书面改为提交成功率 | 确认补充协议条款 |
| 同 Wi‑Fi 期望过高 | 书面说明 IP 隔离必要性 | 认可分 IP 部署方案 |

## 八、验收建议（调试结束后）

### 8.1 建议采用的验收指标（CLI 一期）

1. 功能：配置、登录、试跑、定时预约、日志、PushPlus 可用。  
2. 在 **≥30 个有效测试账号**、**≥3 个申购日**、**已分 IP（≥3 个出口）** 前提下：  
   - **预约提交成功率 ≥ 85%**（剔除已封号、未实名、故意空验证码）。  
3. **不包含**：中签率、付款成功率、统一付款。

### 8.2 不建议采用的指标

- 单次抢购 / 中签 **67%**、**20%**  
- 1000 号 **同一 Wi‑Fi 无代理** 仍要求高成功率
