# 仓库公开后的安全处理清单

若 GitHub 仓库为 **Public**，任何人可克隆代码。已下载的副本**无法撤回**，重点是**立即轮换密钥**并**减少仓库里的敏感信息**。

## 一、立刻做（24 小时内）

| 优先级 | 操作 |
|--------|------|
| P0 | **改服务器 SSH 密码**（腾讯云 ubuntu 用户） |
| P0 | **改 Web 管理密码**：`owner`、`client`（`deploy/.env` 或服务器 `/opt/maotai/deploy/.env`） |
| P0 | **换 `MT_SECRET_KEY`**（≥32 位随机），并同步 `config.yaml` 的 `secret_key` |
| P1 | 改完后 **所有账号重新本机登录**（secret_key 变更会导致旧 Token 失效） |
| P1 | 评估是否将仓库改为 **Private**（Settings → Change visibility） |

## 二、仓库里已暴露的内容（检查）

| 类型 | 现状 | 风险 |
|------|------|------|
| 源代码 | 公开 | 别人可自建一套，不等于能进你的服务器 |
| 服务器 IP | `docs/使用手册.md` 等有 `你的服务器` | 可被扫描、撞库、429 探测 |
| `config.yaml` / `.env` | 已在 `.gitignore` | 若从未误提交则 OK |
| 代理 CSV | 已在 `.gitignore` | OK |
| SSH 密码 | 不应出现在 Git 里 | 若在聊天/文档里出现过，**必须改 SSH** |

## 三、别人下载了会怎样？

- **能做的**：本地运行 CLI/Web、学习代码、自己部署到新服务器  
- **不能直接做的**：没有你的 SSH、owner 密码、`secret_key`、账号 Token，**进不了你已部署的生产环境**（前提是密码没泄露、没弱口令）  
- **若密码弱或写在文档里**：可能被 **撞库登录 Web** 或 **SSH 尝试**

当前仓库约 **8 star / 5 fork / 百次级 clone**，影响面有限，但仍应按泄露处理。

## 四、建议长期做法

1. **仓库改 Private**（客户项目、含部署 IP 的文档）  
2. **文档用占位符**：`http://你的服务器/`、`owner 密码见交付文档`  
3. **生产密钥只放** 服务器 `deploy/.env`，不进 Git  
4. **可选**：Nginx 限制管理端来源 IP；开启腾讯云安全组仅允许办公 IP SSH  
5. README 注明 **个人学习研究**，商业使用与客户数据分离  

## 五、若要把历史里的 IP/密码从 Git 清掉

已 push 的敏感字符串需 **git filter-repo / BFG** 重写历史并 force push（会打断 fork 者）。  
更简单：**轮换所有密码** 比清历史更有效。
