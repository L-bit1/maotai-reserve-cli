# 服务器选购与部署指南

**适用**：茅台抢单软件 Web 管理端 + API +（可选）CLI 定时任务  
**日期**：2026-05-25

---

## 一、先定规模（决定买多大）

| 场景 | 账号数 | 建议套餐 |
|------|--------|----------|
| **个人试用** | 1～10 | 轻量 2C2G |
| **小团队** | 10～100 | 2C4G 或 4C8G |
| **合同/运维** | 100～500 | 4C8G + 代理池 |
| **大规模** | 500～1000 | 4C16G 调度机 + **50+ 代理 IP**（见 [IP隔离部署指南.md](./IP隔离部署指南.md)） |

> **重要**：1000 号瓶颈不在 CPU，而在 **公网出口 IP 数量**。服务器再强，同 IP 堆 1000 号仍会 429。

---

## 二、推荐服务器配置（可直接照抄下单）

### 方案 A：标准生产（推荐，100 号以内 Web + API）

| 项目 | 配置 |
|------|------|
| **类型** | 云服务器 ECS / 轻量应用服务器 |
| **CPU** | 2 核 |
| **内存** | 4 GB |
| **系统盘** | 60 GB SSD |
| **带宽** | 5 Mbps（按量计费亦可，管理端流量不大） |
| **系统** | **Ubuntu 22.04 LTS** 64 位（优先）或 Debian 12 |
| **地域** | 离您办公地近的国内节点（华东/华南） |
| **安全组** | 放行 22、80、443；**不要**对公网开放 8000/3306 |

**参考月费（国内云，2026 行情，活动价浮动）：**

| 厂商 | 约价格 |
|------|--------|
| 阿里云轻量 2C4G | 60～100 元/月 |
| 腾讯云轻量 2C4G | 60～100 元/月 |
| 华为云 / 京东云同类 | 相近 |

---

### 方案 B：多账号 + 定时任务（100～500 号）

| 项目 | 配置 |
|------|------|
| **CPU** | 4 核 |
| **内存** | 8 GB |
| **系统盘** | 80～100 GB SSD |
| **带宽** | 5～10 Mbps |
| **系统** | Ubuntu 22.04 LTS |
| **额外** | HTTP/SOCKS5 代理池（约 20～50 个出口，**另购**） |

**参考月费**：云主机 150～300 元/月 + 代理 数百～数千元/月（视代理商）。

---

### 方案 C：仅 CLI 无 Web（最便宜）

| 项目 | 配置 |
|------|------|
| **CPU** | 2 核 |
| **内存** | 2 GB |
| **系统盘** | 40 GB |
| **用途** | 只跑 `python cli.py` / cron，浏览器本机访问 |

约 **40～70 元/月**（轻量 2C2G）。

## 三、架构示意（服务器上跑什么）

```text
                    用户浏览器 / 后期手机 App
                              │
                              │ HTTPS :443
                              ▼
                    ┌─────────────────┐
                    │  Nginx          │
                    │  /      → 前端   │
                    │  /api/  → API    │
                    └────────┬────────┘
                             │
              ┌──────────────┼──────────────┐
              ▼              ▼              ▼
        web 静态文件    FastAPI:8000    SQLite/MySQL
        (dist/)         (systemd)       (data/)
                             │
                             ▼
                    i茅台官方（出站 HTTPS）
                    多账号时经 proxy_pools 分流
```

**不需要单独买数据库云产品**：100 号以内可用 **SQLite**（项目已支持）；上百号建议 **MySQL 同机 Docker** 或云 RDS。

## 四、购买时注意什么

### 4.1 必选项

- [ ] **国内节点**（访问 i茅台、App Store 版本接口更稳）
- [ ] **Ubuntu 22.04**，不要用 Windows Server（浪费授权费且部署麻烦）
- [ ] 绑定 **SSH 密钥登录**，关闭密码登录（上线后）
- [ ] 购买 **域名**（可选但推荐 HTTPS）

### 4.2 安全组 / 防火墙（只开这些）

| 端口 | 用途 |
|------|------|
| 22 | SSH（建议改非标准端口 + 仅自己 IP） |
| 80 | HTTP（跳转 HTTPS） |
| 443 | HTTPS |

**不要对 0.0.0.0/0 开放**：8000、5173、3306、6379。

### 4.3 与账号规模配套（另购，非服务器配置）

| 账号规模 | 建议出口数 | 说明 |
|----------|------------|------|
| ≤20 | 1（服务器本机 IP） | 可不买代理 |
| 20～100 | 5～10 个代理 | `proxy_pools` 分组 |
| 100～1000 | 20～50 个代理 | 每 20 号一组 |

详见 [IP隔离部署指南.md](./IP隔离部署指南.md)。

## 五、部署步骤概要

### 5.1 本地准备

1. 将项目推到 GitHub / 打包上传到服务器。  
2. 准备好 `config.yaml`（**勿把 secret_key 提交到公开仓库**）。  
3. 域名解析到服务器公网 IP。

### 5.2 服务器初始化（Ubuntu）

```bash
# SSH 登录后
sudo apt update && sudo apt upgrade -y
sudo apt install -y git python3 python3-venv python3-pip nginx certbot python3-certbot-nginx

# Node 20（构建前端用，生产也可在本地 build 后只上传 dist）
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs
```

### 5.3 部署应用

```bash
# 示例目录
sudo mkdir -p /opt/maotai
sudo chown $USER:$USER /opt/maotai
cd /opt/maotai
git clone https://github.com/L-bit1/maotai-reserve-cli.git .
# 或 scp 上传

python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt -r backend/requirements.txt

cp config.example.yaml config.yaml
# 编辑 config.yaml：secret_key、pushplus、proxy_pools 等

# 构建前端
cd web && npm install && npm run build && cd ..

# 环境变量（生产必改）
export MT_ADMIN_PASSWORD='你的强密码'
export MT_SECRET_KEY='随机长字符串'
export MT_CORS_ORIGINS='https://你的域名.com'
```

### 5.4 使用项目自带部署文件（推荐）

仓库内已提供 `deploy/` 目录：

```bash
cd /opt/maotai
cp deploy/.env.example deploy/.env
# 编辑 deploy/.env

# 使用 systemd 管理 API（见 deploy/maotai-api.service）
sudo cp deploy/maotai-api.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable --now maotai-api

# Nginx
sudo cp deploy/nginx.conf.example /etc/nginx/sites-available/maotai
sudo ln -sf /etc/nginx/sites-available/maotai /etc/nginx/sites-enabled/
# 修改其中 server_name、root 路径
sudo nginx -t && sudo systemctl reload nginx

# HTTPS
sudo certbot --nginx -d 你的域名.com
```

### 5.5 定时预约（9:00）

```bash
# 仍可用 CLI cron（服务器时区设为 Asia/Shanghai）
crontab -e
# 每天 8:58 执行（示例）
58 8 * * * cd /opt/maotai && .venv/bin/python main.py >> /opt/maotai/logs/cron.log 2>&1
```

或通过 Web **任务中心** 在 API 内触发（需 API 进程常驻）。

## 七、验收检查清单

- [ ] https://域名 能打开登录页  
- [ ] https://域名/api/v1/ping 返回 `{"code":0,...}`

## 九、常见问题

**Q：买香港/海外节点可以吗？**  
可以访问外网，但 i茅台对异地 IP 可能更敏感；**优先国内大陆节点**。

**Q：要不要买 RDS 数据库？**  
100 号以内 SQLite 够用；上百号、要多机部署时再上 MySQL/RDS。

**Q：2G 内存够吗？**  
仅 API + Nginx + SQLite 勉强够；**建议 4G 起**。

**Q：需要和甲方同一个城市吗？**  
不需要；需要 **稳定出网和足够出口 IP**。
