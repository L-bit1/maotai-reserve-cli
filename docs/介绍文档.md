# 茅台抢单软件 — 产品介绍文档

**版本**：1.0  
**适用平台**：i茅台 App（官方数字营销平台）  
**交付形态**：命令行程序（CLI）  
**开源地址**：https://github.com/L-bit1/maotai-reserve-cli  

---

## 一、产品概述

**茅台抢单软件** 是一款面向个人与小规模使用的 **i茅台预约申购辅助工具**。通过命令行交互界面，帮助用户在每日申购时段内，自动、准时、批量完成 **预约提交**，并管理多账号、收货地址与门店策略。

### 1.1 解决什么问题

| 痛点 | 本软件做法 |
|------|------------|
| 容易忘记 9:00–10:00 申购 | 定时任务，默认提前 2 秒自动提交 |
| 多账号手工操作繁琐 | 支持多账号配置与批量执行 |
| 不知道选哪家门店 | 支持「本市库存最大」「距离最近」等策略 |
| 登录信息难保管 | 本地 AES 加密存储 Token |
| 不确定程序是否正常 | 试跑模式、健康检查、运行日志 |

### 1.2 重要说明（必读）

1. **i茅台 机制是「预约 + 抽签」**，不是电商秒杀。本软件完成的是 **「预约提交」**，**不保证中签**，更不能保证「抢到酒」。  
2. **中签后付款** 须在 **i茅台 App 内** 于规定时间内完成，本软件 **不包含** 自动支付、统一代付。  
3. 请遵守 i茅台 用户协议；勿用于倒卖、外挂、批量违规注册等用途。  
4. 本项目仅供 **学习研究**，使用风险由使用者自行承担。

## 三、工作原理

### 3.1 业务流程

```
用户配置账号/地址/商品
        ↓
每日定时等待至申购窗口（约 9:00 前）
        ↓
校验登录 Token → 获取当日场次 sessionId
        ↓
拉取门店列表 → 按策略选择门店
        ↓
加密预约参数 actParam → 提交预约接口
        ↓
记录日志 / 推送结果
        ↓
（官方抽签）→ 中签用户在 App 内付款提货
```

### 3.2 技术要点

- 调用 i茅台 官方 App 使用的 HTTP 接口（`app.moutai519.com.cn` 等）
- 请求签名（MD5 + 时间戳）与预约体 AES 加密（`actParam`）
- 模拟 iOS 客户端请求头（设备 ID、MT-R、App 版本等）
- App 版本号自动从 App Store 接口同步

### 3.3 与「抢购成功率」的关系

| 指标 | 是否由本软件决定 | 典型量级（经验参考） |
|------|------------------|----------------------|
| **预约提交成功率** | 部分（工程优化） | 账号正常时，大批量账号可达 **较高**（如 85%+） |
| **抽签中签率** | **否**（平台抽签） | 热门款单账号常 **很低**（如 1% 量级或以下） |
| **1000 号至少 1 个中** | 否 | 若单号 p≈1%，概率接近 **必有一个**；但 **≠ 20% 或 67% 账号中签** |

## 五、安装与部署

### 5.1 获取代码

```bash
git clone https://github.com/L-bit1/maotai-reserve-cli.git
cd maotai-reserve-cli
```

### 5.2 安装依赖

```bash
python3 -m venv .venv
source .venv/bin/activate    # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

### 5.3 配置文件

```bash
cp config.example.yaml config.yaml
```

编辑 `config.yaml`，至少修改：

| 配置项 | 说明 |
|--------|------|
| `secret_key` | 任意随机字符串，用于加密本地账号文件 |
| `amap_key` | 高德 Key（可选，用于搜索小区地址） |
| `items` | 预约商品 ID，须与 App 当前商品一致 |
| `shop_strategy` | `max_inventory`（推荐）或 `nearest` |
| `schedule` | 申购时间与提前秒数 |

> `config.yaml` 与 `data/credentials.json` **不会** 提交到 Git，请勿泄露。

### 5.4 启动方式

**方式 A（推荐）**：交互菜单

```bash
python cli.py
```

**方式 B**：macOS 双击

```
启动茅台抢单.command
```

**方式 C**：命令行快捷

```bash
python check.py              # 健康检查
python main.py --dry-run     # 试跑
python main.py               # 正式预约
python main.py --skip-wait   # 跳过等待立即预约
```

### 5.5 定时自动运行（macOS）

参考 `com.imaotai.reserve.plist.example`，配置 launchd 每日 8:58 启动；或使用系统「定时任务」执行 `python main.py`。

## 六、使用指南

### 6.1 首次使用流程

1. 配置 `config.yaml`（`secret_key` 必填）  
2. 运行 `python cli.py`  
3. 菜单选 **「1」** → 填写手机号 → 接收并输入 **短信验证码**  
4. 填写 **收货人、地址**（有高德 Key 可搜索小区）  
5. 菜单选 **「5」** 健康检查  
6. 菜单选 **「3」** 试跑（确认无报错）  
7. 确认无误后，菜单选 **「4」** 或 **「5」** 正式预约  

### 6.2 主菜单说明

| 选项 | 功能 |
|------|------|
| 1 | 账号 / 地址 / 登录 |
| 2 | 试跑预约 |
| 3 | 定时正式预约 |
| 4 | 立即正式预约 |
| 5 | 健康检查 |
| 6 | 查看配置 |
| 7 | 运行日志 |
| 0 | 退出 |

### 6.3 登录说明

- i茅台 **不使用** 传统「账号 + 密码」网页登录。  
- 登录方式为：**11 位手机号 + 短信验证码**（与官方 App 相同）。  
- 可选保存 **支付密码备注**（仅本机加密，供中签后付款参考）。

## 八、常见问题

### Q1：登录出现 HTTP 429？

**原因**：请求过于频繁，触发 i茅台 限流。  

**处理**：

1. 等待 **3–10 分钟** 再试  
2. 不要连续多次点击「发送验证码」  
3. 确保验证码 **完整输入**（4–6 位），不要留空  
4. 关闭 VPN/代理，可换手机热点  

### Q2：sessionId 为 0？

**原因**：当前可能不在申购时段（一般为 **9:00–10:00**）。  

**处理**：在申购时段内运行；试跑可忽略，正式预约会轮询等待。

### Q3：预约成功是否等于中签？

**否。** 预约成功仅表示 **当日申购已提交**；是否中签以 i茅台 App 公布结果为准。

### Q4：多账号能提高中签率吗？

- 多账号 ≈ 多次抽签机会，可提高 **「至少一个号中签」** 的概率。  
- **不能** 把「每个号的中签比例」稳定推到 20%、67% 等。  
- 详见本文 **3.3** 节。

### Q5：商品 ID 在哪里查？

以 i茅台 App 当前商品为准；接口变更时需更新 `config.yaml` 中 `items`。可参考 App 内商品或技术社区开源项目中的 item_code。

## 九、验收建议（面向项目交付）

若用于软件开发合同验收，建议采用 **可验证、可复现** 的指标，例如：

| 建议验收项 | 说明 |
|------------|------|
| CLI 功能完整 | 账号、地址、定时、试跑、日志可用 |
| 预约提交成功率 | 有效测试账号中，成功提交预约的比例（如 ≥85%） |
| 不含中签率承诺 | **不以「抢到酒比例」作为尾款条件** |
| 不含自动付款 | 付款在官方 App 完成 |

**不建议** 将「单次抢购成功率 67%」等条款作为验收标准（与 i茅台 抽签机制不符）。
