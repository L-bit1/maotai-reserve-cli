# 没有安卓手机时，怎么维护 / 逆向 i茅台接口

本 CLI 的请求格式来自 **开源社区 + iOS 抓包**，不强制安卓真机。没有安卓机时，按下面优先级选一条路即可。

---

## 方案一：Mac 上 Android 模拟器（推荐，等同真机）

在 Mac 装 **Android Studio**，用虚拟设备完成抓包和 Frida，无需买手机。

### 1. 安装模拟器

1. 安装 [Android Studio](https://developer.android.com/studio)
2. **Device Manager** → Create Device → 选 Pixel 6 等 → 系统镜像选 **arm64**（Apple 芯片 Mac 用 arm64-v8a 镜像）
3. 启动模拟器，在设置里登录 Google 账号（可选，仅装 App 有时需要）

### 2. 安装 i茅台

- 把 `base.apk.1.1.1` 拖进模拟器窗口，或：

```bash
adb install "/Users/mac/Desktop/工作 /软件/茅台抢单软件/base.apk.1.1.1"
```

### 3. HTTPS 抓包（维护 `crypto.py` / `api.py`）

1. Mac 安装 [mitmproxy](https://mitmproxy.org) 或 Charles  
2. 模拟器 Wi‑Fi 代理指向 Mac IP，端口 `8080`  
3. 浏览器打开 `mitm.it` 安装证书（模拟器需设「用户证书」为信任）  
4. 在模拟器里操作：发验证码 → 登录 → 预约  
5. 把请求里的 `md5`、`timestamp`、`MT-*` 头、`actParam` 与项目对比

若 App 检测代理导致无法联网，可改用 **Postern + VPN 模式** 或只在模拟器里试；失败则用方案二（iPhone）。

### 4. Frida（可选，核对 SALT / SigKey）

```bash
# 模拟器一般是 x86_64 或 arm64，下载对应 frida-server
adb push frida-server /data/local/tmp/
adb shell chmod 755 /data/local/tmp/frida-server
adb shell /data/local/tmp/frida-server &

pip install frida-tools
cd "/Users/mac/Desktop/工作 /软件/茅台抢单软件"
frida -U -f com.moutai.mall -l reverse/frida/hook_sig_key.js --no-pause
```

脚本说明见 `reverse/frida/README.md`。

## 方案二：有 iPhone（最省事）

本工具 **默认模拟 iOS 客户端**（`User-Agent`、`MT-APP-Version` 来自 App Store），与真机 i茅台 **接口通常一致**。

1. Mac 安装 **Proxyman** 或 **Charles**  
2. iPhone 与 Mac 同一 Wi‑Fi，按软件说明装 **描述文件 + 信任证书**  
3. 在 iPhone 上打开 i茅台，走一遍发码 / 登录 / 预约  
4. 在 Mac 上导出请求，对照 `src/api.py`、`src/crypto.py`

若出现 SSL  pinning 无法解密，需要越狱或暂时用模拟器；多数版本 H5/部分接口仍可抓。

---

## 方案三：不做动态逆向（你现在就能用）

APK 静态分析表明：**SALT / AES 与主流开源脚本一致**（见 [APK逆向报告.md](./APK逆向报告.md)）。  
没有真机也可以：

1. 直接跑本仓库 CLI：`python cli.py` → 登录（注意验证码不要空、避免 429）  
2. `MT-APP-Version` 已用 App Store 接口自动拉取，失败时兜底 `1.9.6`  
3. 若登录/预约返回 **401、签名错误、参数错误**，再按方案一或二抓 **一条成功请求** 发维护者改常量即可  

**429** 多为发码太频或验证码为空，不一定是密钥过期。

---

## 方案四：请人代抓一次

只需对方在真机或模拟器上抓 **3 个请求**（发码、登录、预约各 1 条），导出为 HAR 或截图 Header/Body，即可更新配置，无需长期借手机。

---

## 对照表

| 你有啥设备 | 建议 |
|------------|------|
| 只有 Mac | Android Studio 模拟器 + mitmproxy |
| Mac + iPhone | Proxyman 抓 iOS（优先） |
| 什么都没有 | 先用方案三试 CLI；失败再模拟器或请人抓包 |

---

## 和「逆向 APK」的关系

- **静态解 APK**：本仓库已做，但壳加固导致 **抠不出业务 Java**，见 `docs/APK逆向报告.md`  
- **没有安卓机**：用 **模拟器** 或 **iPhone 抓包** 代替，效果一样能维护脚本  
- **Frida**：非必须；抓包足够更新 `SALT` / 请求路径  

个人自用、遵守平台规则；勿用于批量撞库或商用倒卖。
