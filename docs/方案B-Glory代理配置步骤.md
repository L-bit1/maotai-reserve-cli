# 方案 B：天启HTTP / 光络云 固定 IP + Web 发码

适用：云服务器公网 IP 已被 i茅台 429，需通过 **天启HTTP（光络云）固定 IP** 发验证码。

> **品牌说明**：[天启HTTP](https://www.tianqiip.com/) 与 [光络云 Glory](https://www.glorycloud.cn/) 为同一服务商（天启已升级至光络云），**同一账号**登录。  
> 官方 [长效固定套餐操作手册](https://www.tianqiip.com/helpDetails/1560.html)：购买后在 **个人中心 → 我的长效IP / 我的固定IP** 查看 **IP、端口、账号、密码**；程序里写成 `http://账号:密码@IP或网关:端口` 即可（与浏览器填系统代理 + 弹窗输账号密码等价）。

> **客服说「没有隧道」**：固定 IP 产品不叫隧道；你买的 **我的固定IP → 共享IP** 就是 HTTP 代理，CSV 格式正确。  
> 若仍 **412**，优先查 **实名认证** 是否已通过；其次加白名单 `你的服务器`；国内腾讯云是否允许连代理问陈经理 13260802390。

---

## 一、在 Glory 云加白名单（必做）

1. 登录 **Glory 云** 代理商后台（购买 `固定长效IP` 套餐的站点）。
2. 找到 **IP 白名单** / **授权 IP** / **服务器白名单**（名称因平台而异）。
3. 添加你的云服务器公网 IP：

   **`你的服务器`**

4. 保存后等待 **1～5 分钟** 生效。

未加白名单时，代理会报 **412 Proxy Authentication Failed**，Web 发码仍失败。

### 白名单加完后如何自测

在本机项目目录执行：

```bash
python3 scripts/test_glory_proxy.py
```

- 若显示 **1/3 可用** 或更多 → 白名单已生效，继续第二步 `setup_scheme_b.py`
- 若 **0/3 仍 412** → 见下文「仍报 412」

### 国内服务器限制（重要）

Glory 官方文档写明：**使用中国大陆 IP 去连代理** 可能被限制（错误码 **446 Geo-blocked**）。  
你的服务器 `你的服务器` 在**国内腾讯云**时，即使加了 IP 白名单，也可能一直 **412/连不上**。

可向陈经理确认：

- 国内云主机是否允许用「固定 IP / 共享 IP」？
- 若不允许：只能 **换香港/海外服务器** 部署本系统，或 **本机发码**（见 `scripts/local_login_to_server.py`）。

### 仍报 412 时

| 检查项 | 说明 |
|--------|------|
| **实名认证（最常见）** | Glory 要求 **企业/个人实名** 通过后，固定 IP 代理才可用；未实名会表现为 **412**，与白名单、CSV 无关 |
| 加对位置 | **IP 白名单** 里加服务器 **`你的服务器`**（固定 IP 产品，不叫「隧道」） |
| IP 无误 | 与腾讯云公网 IP 一致，无空格 |
| 密码未过期 | 实名通过后，在「我的固定IP」导出最新 CSV → `setup_scheme_b.py` |
| 联系客服 | 陈经理 13260802390：确认实名状态、国内服务器是否可用 |

**实名认证完成后**：等 **10～30 分钟** → Web **系统设置 → 检测全部代理** → 应显示多数 **可用** → 再 **批量发码**。

## 二、一键写入代理池 + 分配出口组（推荐）

在本机项目目录执行（需 SSH 密码）：

```bash
cd "/Users/mac/Desktop/工作 /软件/茅台抢单软件"

# 将 CSV 写入服务器 config.yaml，并为每个账号分配 ip001、ip002…（每号 1 组，适合发码）
DEPLOY_PASS=你的SSH密码 python scripts/setup_scheme_b.py "固定长效IP.csv" --per-group 1
```

脚本会自动：

- 写入 50 条 `proxy_pools`（ip001～ip050）
- 开启 `antidetect.auth_prefer_proxy: true`（发码/登录走代理）
- 为所有启用账号分配 `egress_group`
- 重启 `maotai-api`

---

## 三、在 Web 上验收

1. 用 **owner** 登录：http://你的服务器/
2. **系统设置** → **代理池**
3. 点击 **「检测全部代理」**  
   - 成功：多数显示绿色，提示类似 `发码通路检测：50/50 可用`  
   - 失败：仍显示 `412` → 回到第一步检查白名单
4. 点击 **「为账号分配出口组（每号一组）」**（若脚本已执行可跳过）
5. **账号管理** 或 **批量登录** → 对单个号 **发码** 或 **批量发码**（间隔 ≥90 秒）

---

## 四、账号与出口组对应关系

| 设置 | 说明 |
|------|------|
| `egress_group: ip001` | 使用 `proxy_pools.ip001` 的代理 IP |
| 每号一组（`--per-group 1`） | 发码时各走不同代理，降低限流 |
| 每 20 号一组 | 适合预约：`--per-group 20` |

账号表 **出口组** 列应与 `ip001`… 一致。

---

## 五、常见问题

| 现象 | 处理 |
|------|------|
| 检测代理 412 | Glory 白名单未加 `你的服务器` 或密码过期 |
| 检测代理通过但发码 429 | 该代理出口 IP 也被限流，换组或等 2 小时 |
| 顶部仍提示「服务器公网 IP 429」 | 正常：直连仍被封；**发码走代理成功即可** |
| 代理 0/50 可用 | CSV 账号密码是否过期；在 Glory 续费后重新 `setup_scheme_b.py` |

---

## 六、仅 Web 手工配置（不用脚本）

1. **系统设置** → 代理池 → 粘贴 50 条 URL（或从 CSV 手填）
2. **保存代理配置**
3. **为账号分配出口组**
4. **检测全部代理** → 通过后发码

---

*配置完成后，预约仍走代理；发码/登录优先代理，失败才尝试直连。*
